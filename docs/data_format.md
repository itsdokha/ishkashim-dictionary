# Формат данных словаря (контракт для участников 2 и 3)

Это описание формата файлов словаря. **Договорённость о формате — общая для всей
команды**: участник 2 (поиск) и участник 3 (интерфейс) опираются на эти поля.
Если что-то нужно поменять — сначала обсуждаем, потом я перегенерирую файлы
(данные собираются скриптом `scripts/build_dictionary.py`, руками файлы не правим).

## Какие файлы есть

| Файл | Что это | Кому |
|---|---|---|
| `data/dictionary.json` | **основной формат**, источник правды для кода | участники 2 и 3 |
| `data/dictionary.sample.json` | тестовый набор из 20 статей (та же структура) | для ранней отладки |
| `data/dictionary.csv` | человекочитаемый экспорт (открыть в Excel) | для просмотра глазами |

**В коде используйте JSON.** CSV — только для просмотра; он строится из тех же
данных, но многозначные поля в нём «схлопнуты» в текст.

Все файлы в кодировке **UTF-8**.

## Структура `dictionary.json`

```json
{
  "meta": {
    "language": "ишкашимский язык",
    "version": "1.0",
    "entry_count": 159,
    "pos_legend": { "...": "..." },
    "field_legend": { "...": "..." },
    "sources": [ "..." ]
  },
  "entries": [ { /* статья */ }, ... ]
}
```

* `meta` — служебная информация (можно показать на странице «о словаре»);
* `entries` — массив статей. **Работаем с `entries`.**

## Поля одной статьи

```json
{
  "id": "isk_0013",
  "lemma": "tsām",
  "lemma_lat": "tsam",
  "lemma_cyr": "цам",
  "pos": "noun",
  "pos_ru": "сущ.",
  "pos_full": "существительное",
  "domain": "тело",
  "translations_ru": ["глаз"],
  "translations_en": ["eye"],
  "variants": ["čъm"],
  "examples": [
    { "isk": "Tu χē tsām kūr kun.", "ru": "Сделай свой (собственный) глаз слепым." }
  ],
  "etymology": "Авест. čašman-; ср. тадж. чашм.",
  "comment": "Группа -šm- > -m-. Неодушевл. мн. ч. согласуется с глаголом в ед. ч.",
  "sources": ["Grierson 1920", "Пахалина 1959"]
}
```

| Поле | Тип | Назначение |
|---|---|---|
| `id` | строка | **стабильный уникальный ключ** статьи (`isk_NNNN`). Используйте как ключ в React/списках, в ссылках на карточку. **Не предполагайте, что порядок id = порядок показа.** |
| `lemma` | строка | заголовок в научной латинской транскрипции (с диакритикой). **Показывать пользователю.** |
| `lemma_lat` | строка | то же без диакритики, ASCII (`tsām`→`tsam`). **Главное поле для поиска с клавиатуры.** |
| `lemma_cyr` | строка | приблизительная кириллица (`цам`). Для поиска кириллицей и/или показа. *Это не орфография.* |
| `pos` | строка | **код части речи** — по нему делаем фильтр (стабильные значения, см. ниже). |
| `pos_ru` | строка | краткая помета (`сущ.`) — **показывать** в карточке. |
| `pos_full` | строка | полное название (`существительное`) — для подсказок/легенды. |
| `domain` | строка | тематическая группа (`тело`, `животные`, …) — можно сделать фильтр/категории. Необязательное. |
| `translations_ru` | массив строк | **русские переводы. Главное поле для поиска по-русски.** Показывать через запятую. |
| `translations_en` | массив строк | английские переводы (бонус, по Грирсону). |
| `variants` | массив строк | **варианты написания** (диалекты, иные транскрипции). **Тоже индексировать для поиска** + показывать в карточке. |
| `examples` | массив объектов | примеры: `{ "isk": "...", "ru": "..." }`. Может быть **пустым** — UI должен это переживать. |
| `etymology` | строка | происхождение. Может быть пустой. Показывать в карточке (необязательно). |
| `comment` | строка | лингвистический комментарий, диалектные пометы. Может быть пустым. |
| `sources` | массив строк | источники. Можно показать мелким шрифтом. |

Гарантии по данным (проверяются `scripts/validate_data.py`):
* `id`, `lemma`, `lemma_lat`, `pos`, `pos_ru` — **всегда непустые**;
* `translations_ru` — **всегда минимум один перевод**;
* `pos` — всегда из списка ниже;
* у каждого примера есть и `isk`, и `ru`.

## Коды частей речи (`pos`) — для фильтра участника 2

| `pos` | `pos_ru` | значение |
|---|---|---|
| `noun` | сущ. | существительное |
| `verb` | гл. | глагол |
| `adj` | прил. | прилагательное |
| `adv` | нареч. | наречие |
| `num` | числ. | числительное |
| `pron` | мест. | местоимение |
| `prep` | предл. | предлог |
| `postp` | послел. | послелог |
| `conj` | союз | союз |
| `part` | част. | частица |
| `interj` | межд. | междометие |

Полный список с пометами есть и внутри JSON — `meta.pos_legend` (можно строить
выпадающий фильтр прямо из него, не хардкодя).

## Рекомендации участнику 2 (поиск и логика)

* **Поиск по ишкашимскому слову** — искать в полях `lemma`, `lemma_lat`,
  `lemma_cyr` и в каждом элементе `variants`.
* **Поиск по русскому переводу** — искать в каждом элементе `translations_ru`
  (по желанию — `translations_en`).
* **Нормализация (`normalize.js`):** приводите запрос к нижнему регистру и
  убирайте диакритику. Чтобы поиск работал «как набрали», сравнивайте
  нормализованный запрос с `lemma_lat` (он уже без диакритики). Для русского —
  нижний регистр и, желательно, `ё→е`.
* **Фильтр по части речи (`filters.js`)** — по полю `pos` (стабильные коды).
* **Омонимы.** Есть слова с одинаковым написанием, но это **разные статьи**
  (например, `wak` «один» и `wak` «там»). Поиск должен возвращать **все**
  подходящие статьи (ищите совпадения по всему массиву, а не «первое»).
* **Нестандартные случаи** (для критерия «робастность»):
  пустой запрос → показать всё или подсказку; нет результатов → понятное
  сообщение; разный регистр и диакритика → за счёт нормализации; частичное
  совпадение (подстрока) — желательно.

## Рекомендации участнику 3 (интерфейс)

* Загрузка: `fetch('data/dictionary.json')` → берём `data.entries`.
  (Если откроете `index.html` как файл `file://`, `fetch` может не работать
  из-за CORS — запускайте простой сервер, напр. `python3 -m http.server`.)
* **Карточка слова** — что показать: `lemma` (крупно), `pos_ru`,
  `translations_ru` (через запятую), `variants` («варианты: …»),
  список `examples` (ишк. + перевод), `etymology` и `comment` (если непустые),
  `sources` (мелко).
* `examples`, `etymology`, `comment` **бывают пустыми** — рисуйте блок только
  если есть содержимое.
* Сортировка списка: по `lemma_lat` (латиница) или по `translations_ru[0]`
  (кириллица) — на ваш выбор; на `id` не опирайтесь для порядка.
* `domain` можно использовать для группировки/категорий (необязательно).

## Формат `dictionary.csv`

Одна строка — одна статья, кодировка UTF-8, разделитель — запятая (стандартный
CSV с кавычками). Многозначные поля внутри ячейки разделены ` | `; в примерах
ишкашимская часть и перевод разделены ` :: `. Это **только для чтения глазами**;
для кода берите JSON.

## Версии и изменения

* Текущая версия данных — **1.0** (`meta.version`).
* Если понадобится изменить набор полей или их имена — это **ломающее изменение**;
  сообщаю команде заранее и поднимаю версию.
* Пополнение словаря новыми словами версию полей не меняет — добавляются только
  новые статьи (с новыми `id`).
