# Передача работы — Участник 1 (данные и лингвистика)

Привет! Это моя часть проекта «Учебный веб-словарь ишкашимского языка».
Здесь данные словаря, лингвистическое описание и документация по формату.
Ниже — что сделано и что делать вам.

## Что готово

```
ishkashim-dictionary/
├── data/
│   ├── dictionary.json          ← ОСНОВНОЙ файл данных (159 статей) — используйте его
│   ├── dictionary.sample.json   ← тестовый набор из 20 слов (для ранней отладки)
│   └── dictionary.csv           ← то же в CSV (для просмотра в Excel)
├── docs/
│   ├── linguistic_description.md ← описание языка: что это, зачем, кому полезен (для презентации)
│   └── data_format.md           ← КОНТРАКТ ПО ФОРМАТУ — читать обязательно
├── scripts/
│   ├── build_dictionary.py      ← сборка данных (источник правды; руками data/ не правим)
│   └── validate_data.py         ← проверка целостности данных
└── HANDOFF.md                   ← этот файл
```

* Словарь: **159 выверенных статей** (минимум по плану был 20).
* Источники данных: **Грирсон 1920** (формы, этимологии, примеры из сказки) и
  **Пахалина 1959** (русские значения, современные формы, диалекты).
* Каждая статья: заголовок, транскрипции (научная / ASCII / кириллица),
  часть речи, тема, русские и английские переводы, варианты написания, примеры,
  этимология, комментарий, источники.

## Участнику 2 (поиск и логика)

1. Возьмите для начала **`data/dictionary.sample.json`** (20 слов) — структура
   та же, что и в полном файле.
2. Прочитайте **`docs/data_format.md`** — там по пунктам, в каких полях искать:
   * ишкашимское слово → `lemma`, `lemma_lat`, `lemma_cyr`, `variants`;
   * русский перевод → `translations_ru`;
   * фильтр по части речи → поле `pos` (стабильные коды, список в документе и в `meta.pos_legend`).
3. Для `normalize.js`: поле **`lemma_lat`** уже без диакритики — сравнивайте
   нормализованный запрос с ним.
4. В данные специально заложены **краевые случаи**: омонимы (`wak` «один» / `wak`
   «там»), слова с несколькими переводами, с несколькими вариантами, с пустыми
   примерами — есть на чём проверять робастность.

## Участнику 3 (интерфейс и сборка)

1. Грузите `data/dictionary.json` через `fetch`, берите `data.entries`.
   Для локального запуска поднимите сервер: `python3 -m http.server`
   (через `file://` `fetch` может не сработать).
2. В **`docs/data_format.md`** есть готовый список «что показывать в карточке».
3. Для README и презентации берите факты из **`docs/linguistic_description.md`**
   (ответы на «почему создали» и «кому полезен» — раздел 8; интересные
   лингвистические примеры — раздел 6, особенно слово *rēmuz* «солнце» < Ахура-Мазда).

## Как пересобрать/проверить данные

```bash
python3 scripts/build_dictionary.py   # пересобрать json/csv/sample из скрипта
python3 scripts/validate_data.py      # проверить целостность (должно быть "OK")
```

## Договорённость о формате

Формат полей — **общий контракт**. Если для поиска или интерфейса удобнее другой
формат/имена полей — скажите, я поправлю скрипт и перегенерирую файлы (это быстро).
Главное — не править `data/*.json` руками, иначе при следующей сборке изменения
затрутся.
