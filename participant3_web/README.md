# Ишкашимский словарь — веб-интерфейс

Учебный двуязычный (ишкашимско ↔ русский) веб-словарь ишкашимского языка —
малоресурсного памирского языка иранской группы.

Это **часть участника 3** группового проекта: веб-страница, поиск по словарю
через интерфейс, карточки слов, адаптивная вёрстка и сборка всего продукта
воедино.

![схема](https://img.shields.io/badge/stack-HTML%2FCSS%2FJS-blue) — без фреймворков и сборщиков, чистый статический сайт.

---

## Что это и кому полезно

Словарь предназначен для всех, кому нужен удобный доступ к лексике
ишкашимского языка: студентов-лингвистов, исследователей памирских языков,
носителей и просто любопытствующих. У ишкашимского нет общепринятой
письменности, лексика рассеяна по научным работам — этот словарь собирает её
в одном месте с удобным поиском.

## Возможности

- 🔎 **Поиск** по ишкашимскому слову и по русскому переводу одновременно;
- 🎚️ **Режимы поиска**: везде / только ишкашимский / только русский перевод;
- 🏷️ **Фильтр по части речи** (список строится автоматически из данных);
- 🃏 **Карточка слова**: лемма, помета, переводы (рус./англ.), варианты
  написания, примеры, этимология, лингвистический комментарий, источники;
- ✨ **Подсветка** совпадений в результатах;
- 📱 **Адаптивный интерфейс** — две колонки на десктопе, одна на телефоне;
- 🛟 **Робастность**: пустой запрос показывает весь словарь, «ничего не найдено»
  даёт понятное сообщение, пустые поля статьи не рисуются, нечувствительность
  к регистру и диакритике, корректная обработка омонимов (например, `wak`
  «один» и `wak` «там» — две разные карточки), понятное сообщение при ошибке
  загрузки данных.

## Как запустить

> ⚠️ Сайт читает `data/dictionary.json` через `fetch`. Браузер блокирует `fetch`
> при открытии файла напрямую (`file://`), поэтому нужен **локальный сервер**.

Из **корня проекта** (`ishkashim-dictionary/`):

```bash
python3 -m http.server 8000
```

Затем открой в браузере:

```
http://localhost:8000/participant3_web/
```

Готово — словарь загрузится и сразу покажет все статьи.

### Альтернатива без Python

Любой статический сервер из корня проекта, например:

```bash
npx serve .        # Node.js
php -S localhost:8000   # PHP
```

### Как часть участника 3 связана с остальным проектом

```
data/dictionary.json            (участник 1 — данные)
        │  fetch
        ▼
participant2_js/src/search.js   (участник 2 — поиск, нормализация, фильтры)
        │  import
        ▼
participant3_web/src/app.js     (участник 3 — интерфейс и сборка)
        │
        ▼
```

`app.js` импортирует `searchDictionary` и `getAvailablePartsOfSpeech` напрямую
из папки участника 2 и не дублирует логику поиска. Формат словарной статьи
описан в `../docs/data_format.md`.

## Технические решения

- **Без фреймворков и сборки.** Чистые HTML/CSS/JS + ES-модули — проект легко
  запустить (один `http.server`) и легко защитить.
- **Разделение ответственности.** Интерфейс ничего не знает о том, *как*
  устроен поиск, — он лишь вызывает `searchDictionary(...)` участника 2.
- **Данные — источник правды.** Список частей речи и блок «О словаре» строятся
  из `dictionary.json` (`meta.pos_legend`, `meta.sources` и т. д.), ничего не
  хардкодится.
- **Безопасность вывода.** Все значения из данных экранируются (`escapeHtml`),
  подсветка совпадений тоже работает по экранированному тексту.
- **Отзывчивость.** Ввод в поиск дебаунсится (150 мс), чтобы не пересчитывать
  выдачу на каждую букву.

## Краевые случаи, которые обрабатывает интерфейс

| Ситуация | Поведение |
|---|---|
| Пустой запрос | показывается весь словарь |
| Нет результатов | понятное сообщение с подсказкой сменить слово/режим |
| Пустые `examples` / `etymology` / `comment` | блок просто не рисуется |
| Омонимы (`wak`) | показываются все совпавшие статьи |
| Разный регистр и диакритика | нормализуются (логика участника 2) |
| Выбранное слово выпало из новой выдачи | карточка сбрасывается |
| Не удалось загрузить данные | заметное сообщение с инструкцией про сервер |

## Известные ограничения

- Это **прототип**: транскрипция упрощена и унифицирована, орфографии у языка
  нет.
- Поиск — по подстроке без морфологии: словоформы (склонение/спряжение) не
  приводятся к начальной форме.
- Объём — 159 статей (базовая лексика), это учебный, а не полный словарь.
